=== Subject of the review ===

A new unified Settings system inside `@my-own-web-services/react-components`.
The goal is: every persisted setting (theme, language, hotkey overrides,
recent actions, and any consumer-app-registered fields) lives in a single
localStorage blob under `${storagePrefix}_settings`, structured as
`{ _v, core: {...}, app: { <appKey>: {...} } }`. Apps register their own
typed settings via `defineAppSettings({ appKey, schema })`, read+write via
`useAppSetting(schema, "fieldId")`, and SettingsPanel renders the registered
schema as additional sections. Legacy per-concern keys (`_theme`, `_language`,
…) auto-migrate on first mount. The future story is a remote-sync provider
for the `core` slot, while `app.*` stays device-local — so the persistence
layer is an interface (`SettingsStorageAdapter`) for swap-in later.

Focus your review on: this Settings system. Skip anything labelled
out-of-scope above.
